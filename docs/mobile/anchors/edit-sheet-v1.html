Mobile Edit Sheet — v1 (LOCKED)

Reference design for mobile-edit-sheet.tsx + mobile-rich-text-editor.tsx (Phase 2 T2-4b consolidated). One editor pattern serves three Daily Report consumers: Daily Recap, Daily Thoughts, Market Notes.
  • MobileEditSheet: generic fullscreen-style sheet with drag handle, X / title / Save header, dirty-dismiss confirm (iOS Mail action-sheet style)
  • MobileRichTextEditor: contentEditable + 15-surface toolbar (Bold / Italic / Underline | Heading | Bullet / Numbered / Indent / Outdent | Color / Link | Quote / Code | Clear | Undo / Redo), horizontal-scroll single-row, sticky-above-keyboard via visualViewport observer
  • Color picker: tap palette icon → 5 swatches (TAG_PALETTE: rose / amber / emerald / sky / violet) + dashed default-reset. Class-based .text-color-X (mirrors .indent-N; no DOMPurify config change)
  • DOMPurify: identical paste sanitizer to desktop ThoughtsEditor; image paste stripped (Captures section handles attachment)
  • Consumers: Daily Recap → lowlights (HTML). Daily Thoughts → daily_thoughts (HTML). Market Notes → market_notes (plain textarea, no rich editor — one-liner)
  • Save: POST /api/journal/edit with the matching field name. localStorage draft per field cleared on success or clean dismiss; preserved across dirty-confirm flow.
  • Daily Report integration: Recap + Thoughts get markdown-rendered previews (ReactMarkdown + rehypeRaw) with Edit pill; Market Notes one-liner sits in header subtitle area with pencil affordance; metrics tile relabeled DAILY → DAILY P&L; desktop daily-report-card.tsx gets a parallel Market Notes edit affordance.
